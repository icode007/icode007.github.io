基于Github 和 hexo 搭建个人博客网站

看到很多大牛都有自己的个人网站，觉得很高端。今天仔细的看了下网上关于搭建个人博客网站的教程，发现使用别人的工具来构建一个自己的网站并不难，下面介绍如何以来github和hexo框架来完成。

Github

Github可以说是程序员的神器，关键还免费。我们搭建个人网站就是基于Git上提供项目托管，而我们可以把博客项目放在Github上，然后通过在地址栏输入（owner名字.github.io）owner_name.github.io来访问博客主页。

也就是说github提供了服务器加域名的功能，当然你也可以购买自己的.com,.cn等域名，只要把域名双向绑定就可以了。

也可以购买服务器服务，当然那就不一定使用hexo框架了。

设置 Github

  • 安装Git

  • 到Github官网使用邮箱注册一个github账号。输入用户名，密码。验证，ok(已有github账户可跳过)。
    Github官网

  • 新建一个代码仓库(repository)，设置repository name 等于ownername.github.io (ownername即为注册时名字).

Hexo

hexo其实就是一个基于Node.js的框架，将source文件夹下的.md和.html文件生成html文件。 然后使用git推送到github上。而html文件能被github pages解析。

安装，使用hexo

  • 安装Node.js Node.js

  • 使用node的npm包管理工具安装hexo。注意：由于国内网络使用npm安装包速度一般较慢，所以可以使用一些镜像网站。比如taobao的镜像网站。淘宝npm镜像，这样在使用npm时可以用cnpm命令代替（可以查看说明文档）。

$ npm install -g cnpm --registry=https://registry.npm.taobao.org

  • 使用npm/cnpm安装 hexo-cli，-g全局安装。
    npm install hexo-cli -g

  • 安装了hexo后，我们就可以使用hexo命令来构建个人博客项目了。

  • 配置项目下的_config.yml文件，配置title，deploy等各种配置。最重要的配置是：

上面repository对应的值为github上对应项目的ssh链接。

npm install hexo-deployer-git --save //安装这个自动部署发布工具

  • 在source/_posts文件夹下新建.md文件，然后使用markdown写作。
    hexo server 或 hexo s//启动本地服务器地址栏输入(https://localhost:4000 )查看效果

  • 最后发布到我们的github pages

  • 以后每次写完md，使用命令
    hexo clean && hexo g && hexo d //上面命令的缩写发布即可。
